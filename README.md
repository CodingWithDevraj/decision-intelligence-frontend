# Decision Intelligence Platform - Frontend

This repository contains the frontend application for the Decision Intelligence Platform. It provides an interactive dashboard that visualizes analytics, simulations, forecasts, and optimized business strategies generated by the backend system.

## Overview

The frontend is a React-based single-page application that enables users to interact with a decision intelligence system. It consumes REST APIs to present insights, simulate scenarios, and support data-driven decision-making.

The UI is designed to be modular, interactive, and responsive, with a focus on clarity and usability.

---

## Core Features

### Data Upload Interface
- Upload structured datasets through the UI
- Triggers backend ingestion and processing

### Insights Dashboard
- Displays key metrics such as total revenue, top product, and best-performing region

### Decision Recommendations
- Presents actionable suggestions derived from backend analytics

### Simulation Engine
- Allows users to adjust price and demand variables
- Updates revenue and profit dynamically based on input

### Strategy Optimization
- Displays the optimal price-demand configuration for maximum profit

### Root Cause Analysis
- Highlights underperforming products and regions

### Forecasting
- Shows predicted revenue based on current trends

### Scenario Comparison
- Visual comparison of multiple strategies using charts

## Application Architecture

The frontend follows a component-based architecture:

src/
- components/
  - Upload.js
  - Insights.js
  - Decisions.js
  - Simulation.js
  - Strategy.js
  - RootCause.js
  - Forecast.js
  - Compare.js
  - NotesSection.js
  - WorkflowSection.js
- App.js
- App.css

Each component is responsible for a specific feature and communicates with backend APIs independently.

---

## Data Flow

1. User uploads dataset via UI
2. Backend processes and stores data
3. Frontend sends API requests to retrieve results
4. Responses are rendered as visual components and charts
5. User interactions (e.g., simulation sliders) trigger dynamic API calls

---

## API Integration

The frontend interacts with the following backend endpoints:

- GET /api/insights
- GET /api/decisions
- GET /api/simulate
- GET /api/strategy
- GET /api/root-cause
- GET /api/forecast
- GET /api/compare
- POST /api/upload

All API calls are handled using the Fetch API and are dynamically triggered based on user actions.
